Key confusions — most commonly tested
RCF vs IP InsightsRCF = any anomaly in any data. IP Insights = IPv4 address behavior anomalies SPECIFICALLY.
LDA vs BlazingTextLDA = discover UNKNOWN topics (unsupervised, no labels). BlazingText = classify into KNOWN categories (supervised, needs labels).
Object Detection vs Image ClassificationCOUNT / LOCATE / MULTIPLE objects = Object Detection (bounding boxes). ONE label for entire image = Image Classification.
Seq2Seq use casesMachine translation, text summarization, speech-to-text, caption generation. Input sequence → output shorter/different sequence.
XGBoost regression HPO metricUse validation:rmse. NOT validation:accuracy (classification metric), NOT validation:ndcg (ranking metric).
Object2VecLearns vector embeddings of discrete objects (user IDs, item IDs). NOT for images. NOT for text classification.
Metric selection decision tree
Continuous output (house price, car value)MAE or RMSE — regression only
Time-series forecasting (temperatures, humidity)RMSE + Average wQL
Minimize false negatives (missed cancer, missed fraud)Maximize Recall = TP ÷ (TP + FN)
Minimize false positives (false spam, over-treatment)Maximize Precision = TP ÷ (TP + FP)
Balance precision and recallF1 Score — harmonic mean
Overall correct predictions (spam + not-spam both count)Accuracy
Model ranking ability across all thresholdsAUC-ROC
Binary classification (Passed/Failed, spam/not-spam)Precision + Recall + Accuracy + F1 — ALL FOUR valid
Fraud: detect fraud + minimize false flagsFalse Positive Rate + F1 Score
Reduce FP in Linear Learner classifierIncrease target_precision hyperparameter
Text summarization qualityROUGE score — summarization ONLY
Classification probability output qualityLog Loss
Never-mix rules — hardcoded
RMSE / MAE = regression ONLYNever for Passed/Failed, spam/not-spam, fraud classification problems.
Accuracy / Precision / Recall / F1 = classification ONLYNever for continuous outputs like house prices or car values.
ROUGE = text summarization ONLYNever for classification, regression, or image tasks.
Cut-off / threshold = NOT a metricDecision boundary only. Never select as a performance evaluation metric answer.
AUC = binary classification ranking onlyNot for regression, not for time-series forecasting.
Average wQLProbabilistic forecasting — predicts a distribution/quantiles, not just a point value. Time-series like weather, demand. Always pairs with RMSE.
Explainability techniques
PDP (Partial Dependence Plot)How does predicted probability change as ONE feature increases? (e.g. churn probability vs customer age). Shows feature trend.
DPL (Difference in Proportions of Labels)Label proportion imbalance across demographic groups. Group A churn rate 40% vs Group B 10% → DPL flags this disparity.
SHAP / Shapley valuesFeature importance for a trained model's predictions. Requires a trained model. Used by Clarify and ModelExplainabilityMonitor.
TVD / KL DivergenceDistribution distance metrics. NOT explainability techniques for demographic fairness on MLA-C01.

Deployment strategies
Blue/GreenInstant full traffic switch. Easiest rollback. Full duplicate environment required.
CanarySmall % of traffic first (e.g. 5%). Gradually shift all. Safe rollout. Catch issues early.
A/B testingPersistent traffic split between model variants. Compare accuracy, revenue impact over time.
ShadowMirror live traffic to new model, zero user impact. Silent comparison before switching.
Inference parameters
TemperatureLower = deterministic/consistent. Higher = random/creative. For standardized investment advice → lower temperature.
Top KLimits candidates to top K tokens. Lower K = more focused output. Higher K = more diverse. All three go same direction.
Top P (nucleus sampling)Cumulative probability cutoff. Lower P = focused. Higher P = diverse. Decrease ALL THREE for max consistency.
Mixed parameter directionsIncrease Temp but decrease Top K = they partially cancel out. All three must move same direction for clear effect.

Top 20 exam traps — compiled from all three practice sets
TRAP 1 — Reserved Instances + SageMaker trainingNOT supported. Exam offers this every single time as a cost distractor. Always use Savings Plans for long-term SageMaker cost savings.
TRAP 2 — DataBrew + mixed file types in one folderFails. You cannot mix CSV + JSON + Parquet in one folder. Correct = separate folders first, then process each folder, output as Parquet.
TRAP 3 — Rekognition + audio or speechRekognition = visual content analysis ONLY (images, video frames, faces, objects). For spoken speech in video → Transcribe. Rekognition cannot hear.
TRAP 4 — ModelExplainabilityMonitor vs ModelBiasMonitorFeature importance RANK shifted (age was #1, now driving history #1) = Explainability Monitor (SHAP drift). Bias across demographic groups = Bias Monitor.
TRAP 5 — SSE-KMS AccessDenied — fix the execution role, not the userTraining jobs use execution role credentials. Add kms:Encrypt + kms:Decrypt to the EXECUTION ROLE. The IAM user's permissions are irrelevant.
TRAP 6 — Gateway endpoint + cross-regionGateway VPC endpoint = same-region S3/DynamoDB ONLY and it's free. Cross-region = Interface endpoint (PrivateLink). Never mix these up.
TRAP 7 — Bedrock fine-tuning dataset already existsIf question says "already has JSONL dataset" or "already has labeled data" — SKIP the Prepare step. Start directly at fine-tuning job.
TRAP 8 — Provisioned Throughput = mandatory last step"Use the customized model in Bedrock tasks" sounds like the final step. Wrong. Buy Provisioned Throughput first, THEN use the model.
TRAP 9 — Cut-off is NOT a metricA threshold/cut-off is a decision boundary, NOT an evaluation metric. Never select it when asked for performance metrics.
TRAP 10 — Pipe Mode + NLP tokenizersPipe Mode cannot seek within files — sequential only. This breaks NLP tokenizers needing random access. Use Fast File Mode for BERT/NLP.
TRAP 11 — True Positive Rate ≠ metric for minimizing false positivesFraud question asks to minimize false positives → use FPR + F1. NOT True Positive Rate (TPR = recall = minimizing false negatives).
TRAP 12 — AWS Budgets cannot control executionBudgets only sends alerts when spending exceeds a threshold. Cannot pause training jobs, adjust schedules, or control SageMaker execution.
TRAP 13 — RMSE for classification problemsRMSE is REGRESSION ONLY. Never use for Passed/Failed, spam/not-spam, fraud/not-fraud, or any discrete label prediction.
TRAP 14 — Adding max_depth to fix overfittingMinimizing max_depth reduces tree complexity and overfitting in XGBoost. ADDING max_depth makes overfitting worse. Never increase to improve generalization.
TRAP 15 — k-NN worker sample splitsample_size=200, 2 workers → each worker gets 100 points (200 ÷ 2). NOT 200 points each. NOT the total dataset size.
TRAP 16 — Model Monitor baseline after model updateModel updated → old baseline is now stale → generate a NEW baseline from updated dataset → reconfigure Model Monitor to use new baseline.
TRAP 17 — Cost Allocation Reports auto-generate tagsFALSE. Cost Allocation Reports do not auto-generate tags. They only read tags you've already created and activated. "Automatically generate" = wrong.
TRAP 18 — Comprehend ImportModel for cross-account model copyNOT S3 export, NOT CloudFormation, NOT AWS Direct Connect. Use ImportModel API + resource-based IAM policy in source DEV account.
TRAP 19 — Polly → Transcribe for text dataConverting text to speech (Polly) then back to text (Transcribe) is circular and wasteful. For text data → process directly with Translate and LLM.
TRAP 20 — LDA vs BlazingText for "unknown topics"If topics are UNKNOWN and need to be discovered automatically = LDA (unsupervised). If topics are known and predefined = BlazingText classification (supervised).
