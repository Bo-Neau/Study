VPC endpointsGateway = same-region S3/DynamoDB, FREE. Interface (PrivateLink) = cross-region + inter-service (SageMaker→Comprehend). Private subnet + interface endpoint = correct fix for VPC endpoint connection failures.

Top 30 exam traps — all 5 practice sets
TRAP 1 — Reserved Instances + SageMakerNOT supported. Always wrong. Use Savings Plans.
TRAP 2 — DataBrew + mixed files in one folderFails. Separate folders first. Each folder same format + same extension.
TRAP 3 — Rekognition + audio/speechVisual ONLY. Speech → Transcribe.
TRAP 4 — ModelExplainabilityMonitor vs ModelBiasMonitorSHAP rank shifted = ExplainabilityMonitor. Bias across groups = BiasMonitor.
TRAP 5 — SSE-KMS fix = execution role, not IAM userkms:Encrypt+Decrypt → EXECUTION ROLE. S3 GetObject error → s3:GetObject+s3:ListBucket on execution role.
TRAP 6 — Gateway endpoint + cross-regionGateway = same-region S3/DynamoDB only. Cross-region = Interface endpoint.
TRAP 7 — Bedrock dataset already existsSkip "Prepare JSONL" step. Start at fine-tuning job.
TRAP 8 — Provisioned Throughput is mandatoryCannot invoke fine-tuned Bedrock model without buying PT first.
TRAP 9 — Cut-off is NOT a metricDecision boundary only. Never select as evaluation metric.
TRAP 10 — Pipe Mode + NLPCannot seek. Breaks NLP tokenizers. Use Fast File Mode.
TRAP 11 — TPR ≠ metric for false positivesMinimize FP → Precision. Fraud FP+FN balance → FPR+F1.
TRAP 12 — Budgets cannot control executionAlert only. Cannot pause jobs.
TRAP 13 — RMSE for classificationRegression ONLY. Never for binary/class prediction.
TRAP 14 — Adding max_depth for generalizationMinimizing max_depth reduces overfitting. ADDING makes it worse.
TRAP 15 — k-NN sample_size splitsample_size=200, 2 workers → 100 each (200÷2).
TRAP 16 — Model Monitor stale baselineModel updated → generate NEW baseline → reconfigure Monitor.
TRAP 17 — Cost Allocation Reports auto-generate tagsFALSE. They read already-created activated tags.
TRAP 18 — Comprehend ImportModel cross-accountImportModel API + resource-based IAM in DEV account. NOT S3, NOT CloudFormation, NOT Direct Connect.
TRAP 19 — Underfitting fix = regularizationUnderfitting = add features, REDUCE regularization. NOT dropout (overfitting fix).
TRAP 20 — Learning rate oscillation = increase LROscillating loss = LR too HIGH. Fix: REDUCE LR. NEVER increase.
TRAP 21 — AutoPilot for K-MeansAutoPilot = supervised ONLY. K-Means HPO → AMT.
TRAP 22 — Warm start IDENTICAL when data changesNew data = TRANSFER_LEARNING. Same data = IDENTICAL_DATA_AND_ALGORITHM.
TRAP 23 — EventBridge IAM = SendPipelineExecutionStepSuccessEventBridge needs StartPipelineExecution to TRIGGER pipeline. SendPipelineExecutionStepSuccess = status update only.
TRAP 24 — Confusion matrix for threshold selectionUse ROC to find optimal threshold. Confusion matrix = single threshold snapshot only.
TRAP 25 — CloudTrail management events for Bedrock auditBedrock agent/KB interactions = DATA events (not management). Management events = high-level ops only.
TRAP 26 — Lambda for large video inference (>6MB)Lambda = 6MB limit. 600MB video files → Async Inference. Lambda CANNOT handle large ML payloads.
TRAP 27 — Scatter plot for regression evaluationScatter plot = relationship between 2 features (EDA). Regression over/under-estimation = RESIDUAL PLOT.
TRAP 28 — Production variants = multiple endpointsProduction variants work on ONE endpoint only. NOT across multiple endpoints.
TRAP 29 — Kendra for text comprehension/entity extractionKendra = semantic SEARCH on documents. Comprehend = NLP analysis (entities, PII, sentiment). Different tools.
TRAP 30 — Serverless Inference = Asynchronous InferenceServerless = low latency, auto-scale, per-request billing (use Provisioned Concurrency for warm). Async = queued, up to 1GB, up to 15 min, scales to zero. Completely different.
