Algorithm selection decision tree
Tabular classification or regressionXGBoost
Fast linear baselineLinear Learner
Cluster/segment data (no labels)K-Means (unsupervised)
Anomaly in any numerical/time-series dataRandom Cut Forest (RCF)
Anomaly in IPv4 address patterns specificallyIP Insights (IPv4 only)
Discover UNKNOWN topics (no labels)LDA — Latent Dirichlet Allocation
Classify into KNOWN text categories (supervised)BlazingText — Text Classification mode
Word embeddings / Word2VecBlazingText — Word2Vec mode
Text → shorter text (summarize/caption/translate)Seq2Seq
Time series forecasting (RNN-based)DeepAR
Recommendations on sparse dataFactorization Machines
Remove weak features / dimensionality reductionPCA
COUNT or LOCATE multiple objects, bounding boxesObject Detection — MXNet (SSD)
One label for entire imageImage Classification (CNN)
Label every pixel in imageSemantic Segmentation
CNN training (custom framework)TensorFlow or PyTorch script mode
k-NN: sample_size=200, 2 workers100 points per worker (200÷2)
Key algorithm confusions
RCF vs IP InsightsRCF = any anomaly. IP Insights = IPv4 behavior anomalies SPECIFICALLY.
LDA vs BlazingTextLDA = discover UNKNOWN topics (unsupervised). BlazingText = KNOWN categories (supervised, needs labels).
Object Detection vs Image ClassificationCOUNT / LOCATE / MULTIPLE = Object Detection. ONE label for whole image = Image Classification.
XGBoost HPO for imbalanced classificationImbalanced binary classification → validation:f1. Regression → validation:rmse. Ranking → validation:ndcg.
Metric selection decision tree
Continuous output (house price, car value)MAE or RMSE — regression only
Time-series forecasting (temperature, humidity)RMSE + Average wQL
Minimize false negatives (missed cancer, fraud)Maximize Recall = TP÷(TP+FN)
Minimize false positives (false spam, over-treatment)Maximize Precision = TP÷(TP+FP)
Minimize FP specifically (falsely flagging legit transactions)Precision alone
Balance precision and recallF1 Score — harmonic mean
Imbalanced binary classification HPOvalidation:f1 (accuracy misleading on imbalanced)
Overall correct predictions (spam + not-spam both)Accuracy
Fraud: detect fraud + minimize false flagsFalse Positive Rate + F1 Score
Find optimal classification thresholdROC curve (TPR vs FPR at various thresholds)
Regression model over/under-estimation checkResidual plot (residuals vs predicted values)
Text summarization qualityROUGE score — summarization ONLY
Never-mix rules
RMSE / MAE = regression ONLYNever for Passed/Failed, spam, fraud classification. Exam loves offering RMSE for classification.
Accuracy/Precision/Recall/F1 = classification ONLYNever for continuous outputs like house prices.
ROUGE = text summarization ONLYNever classification, regression, or image tasks.
Cut-off / threshold = NOT a metricDecision boundary only. Never select as a performance metric.
Confusion matrix = snapshot, not threshold selectorShows TP/TN/FP/FN at ONE threshold. Use ROC to find the BEST threshold across all levels.
Residual plot vs scatter plotResidual plot = regression error analysis (under/over-estimation). Scatter plot = relationship between two features. Don't confuse for regression evaluation.

TRAP 2 — DataBrew + mixed file types in one folderFails. Separate folders first. Process each separately. Output = Parquet.
TRAP 3 — Rekognition + audio/speechVisual ONLY. Speech in video → Transcribe. Rekognition cannot hear.
TRAP 4 — ModelExplainabilityMonitor vs ModelBiasMonitorFeature importance rank shifted = Explainability (SHAP drift). Bias across groups = Bias Monitor.
TRAP 5 — SSE-KMS fix = execution role, not IAM userkms:Encrypt + kms:Decrypt → EXECUTION ROLE. Jobs use execution role credentials, not user credentials.
TRAP 6 — Gateway endpoint + cross-regionGateway = same-region only. Cross-region = Interface endpoint (PrivateLink).
TRAP 7 — Bedrock fine-tuning dataset already existsSkip "Prepare JSONL" step. Start at fine-tuning job directly.
TRAP 8 — Provisioned Throughput = last step before invokingCannot invoke custom Bedrock model without PT. "Use the model" is NOT the final step.
TRAP 9 — Cut-off is NOT a metricDecision boundary only. Never select as performance evaluation metric.
TRAP 10 — Pipe Mode + NLPCannot seek. Breaks NLP tokenizers. Use Fast File Mode for BERT/NLP.
TRAP 11 — True Positive Rate ≠ metric for false positivesMinimize FP → Precision. FPR + F1 for fraud. TPR = recall = minimizing FN.
TRAP 12 — AWS Budgets cannot control executionAlert only. Cannot pause jobs or manage schedules.
TRAP 13 — RMSE for classificationRMSE = regression ONLY. Never for binary/multi-class classification.
TRAP 14 — max_depth fix for overfittingMinimize max_depth = simpler trees = less overfitting. ADDING max_depth = worse overfitting.
TRAP 15 — k-NN sample_size splitsample_size=200, 2 workers → 100 each (200÷2). NOT 200 each.
TRAP 16 — Model Monitor baseline after model updateOld baseline is stale. Generate NEW baseline → reconfigure Monitor.
TRAP 17 — Cost Allocation Reports auto-generate tagsFALSE. They read already-created activated tags.
TRAP 18 — Comprehend ImportModel cross-accountImportModel API + resource-based IAM in DEV account. NOT S3, NOT CloudFormation, NOT Direct Connect.
TRAP 19 — Underfitting fix ≠ regularizationPoor on BOTH train and val = underfitting. Fix: add domain-specific features. NOT increase regularization (makes worse). Overfitting = Dropout.
TRAP 20 — Auto-scaling cooldown vs scaling typeToo-frequent scaling = cooldown period. Predictable spikes = scheduled scaling. NOT increase instance type.
TRAP 21 — SageMaker Debugger vs TensorBoardReal-time gradient anomalies + class imbalance alerts during training = Debugger. Layer visualization for CNN misclassification = TensorBoard.
TRAP 22 — Warm start type selectionNEW data = TRANSFER_LEARNING. Same data + same algo = IDENTICAL_DATA_AND_ALGORITHM. Exam will give both options.
TRAP 23 — EventBridge IAM permission for PipelinesStartPipelineExecution = trigger pipeline. SendPipelineExecutionStepSuccess = status update. Never confuse the two.
TRAP 24 — @step vs @remote decoratorAdd custom Python to SageMaker Pipelines = @step decorator. @remote = runs outside managed pipeline environment.
TRAP 25 — Confusion matrix vs ROC curveROC = find optimal threshold. Confusion matrix = evaluate at ONE specific threshold. To SELECT best threshold = ROC.
